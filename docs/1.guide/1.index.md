# Getting Started

> Learn how to install and set up the Claude Code Validator framework in your project.

1. Install Package

   ::note
   Please note that steps only work with numbered lists and is not within children.
   ::

   :pm-install{name="-D claude-code-validator"}

2. Set up hooks

   ```json [.claude/settings.json]
    {
      "hooks": {
        "PreToolUse": [
          {
            "matcher": "Edit",
            "hooks": [
              {
                "type": "command",
                "command": "bunx claude-code-validator validate",
                "timeout": 10
              }
            ]
          },
          {
            "matcher": "Write",
            "hooks": [
              {
                "type": "command",
                "command": "bunx claude-code-validator validate",
                "timeout": 10
              }
            ]
          }
        ]
      }
    }
   ```

2. Create Your First Rule

   ```ts [.claude/rules/my-rule.ts]
   import { defineCodeRule } from "claude-code-validator";

   export const myRule = defineCodeRule({
     name: "my-rule",
     description: "Prevents usage of console.log",

     shouldRun: (context) => {
       // Only run on TypeScript and JavaScript files
       return /\.(ts|js)x?$/.test(context.filePath);
     },

     validate(context) {
       const errors: string[] = [];

       if (/console\.log/.test(context.content)) {
         errors.push(

             `   â†’ Use a proper logging library instead\n` +
             `   ğŸ“„ File: ${context.filePath}`
         );
       }

       return errors;
     },
   });
   ```

## Test Auto-Discovery

Verify your rule is discovered:

```bash
bun .claude/claude-code-validator/src/cli.ts list-rules
```

You should see output like:


## Test the Integration

Try validating manually:

```bash
echo '{"tool_name":"Write","tool_input":{"file_path":"test.ts","content":"console.log(\"test\")"}}' | \
  bunx claude-code-validator validate
```

You should see a validation error blocking the operation!
